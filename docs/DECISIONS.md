# 決定事項（不変の原則）

この文書は、設計の土台となる「変えない原則」だけを書く。細部の仕様や運用は他ドキュメントで検討・更新する。

## 原則
1. **ドキュメントが正本**
   - Controlled Spec（自然言語中心の DSL 的入力）を唯一の正本とする。
   - ドメイン研究者が読んで理解でき、かつ実行モデルへ**決定的に変換可能**であることを両立する。
2. **妥当性保証は出口で行う**
   - AI/LLM の自由度・非再現性・ハルシネーションは前提として受け入れる。
   - 実行結果の妥当性は、**出口での物理的検証**によって保証する。
   - bitwise 一致は要求しない。
