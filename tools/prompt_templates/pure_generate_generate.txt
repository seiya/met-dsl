Authoring rules (the deterministic gates your bundle must pass ‚Äî the host runs them on your source AFTER you return, and you get no second look at them, so satisfy them on the first emission).
(1) Style lint (`Generate.lint`, fortitude): keep every source line at or under 100 columns (split with a trailing `&` continuation); import with an explicit `use <module>, only: <names>` list ‚Äî never a bare `use`; import intrinsic modules as `use, intrinsic :: iso_fortran_env, only: ...`; declare every real/integer with a NAMED kind from `iso_fortran_env` (`real64`, `int32`) ‚Äî never `real(8)` or a bare `real`, and remember that adding this import is itself subject to the `use, intrinsic ::` rule above; give every `select case` a `case default` arm.
(2) The `implicit none` trap ‚Äî this exact pair of mistakes cost a prior run its whole retry budget, so read it twice. The lint wants `implicit none` to carry the F2018 spec-list `implicit none (type, external)`, but the build runs `-std=f2008`, where that spec-list is a COMPILE ERROR. Do NOT "fix" one by introducing the other; do NOT oscillate between them. The ONE correct form is a plain `implicit none` with a suppression directive on its OWN LINE immediately above it: a line containing only `! allow(C003)`, then the next line `implicit none`. Putting the directive on the SAME line as `implicit none` is wrong twice over (the C003 finding survives AND you earn a FORT002 finding). Writing `implicit none (external)` or `implicit none (type, external)` is wrong (compile error under `-std=f2008`). Emit the two-line form everywhere and move on.
(3) Syntax gate (`Generate.syntax` ‚Äî a real `gfortran -fsyntax-only -std=f2008` front-end run, so a `! allow(...)` directive CANNOT suppress anything here): identifiers must be 63 characters or shorter; `stop` and `error stop` accept only a CONSTANT expression as their code/message ‚Äî to report a runtime value, `print` it first and then `error stop` with a literal; the gate promotes `-Werror=unused-dummy-argument` and `-Werror=unused-variable`, so an ABI-fixed dummy you genuinely do not use must be BOUND with `associate (unused_<name> => <name>)` / `end associate` around a statement (an arithmetic no-op that "uses" it is not acceptable), while an unused LOCAL must simply be deleted; Fortran identifiers are case-INSENSITIVE, so `Flux` and `flux` collide ‚Äî when a local or dummy would collide with an IR-derived name, keep the IR name verbatim and rename YOUR side.
(4) Dataflow gate (`Generate.static`): an operation's input state is `intent(in)`, its updated state is `intent(out)`, and EVERY `intent(out)` dummy must be reached by an assignment chain before the procedure returns ‚Äî an `intent(out)` you never assign fails the gate.

If and ONLY IF there is no sink, the call is INERT: assign every actual argument before the call and let the result flow nowhere. Do NOT invent a purpose for an inert call ‚Äî no probes, no thresholds, no validity flags, no branching on its result.
